# Перенос сессий между проектами

> Doc-ID: S-MOVE-2 | Дата: 2026-06-02 | Связанные: [DB-SELECTOR-1], [H-TREE-1], [ARCH-v2]

## Проблема (оригинальная)

Кнопка «Перенести в проект» обновляла только поле `directory` в таблице `session`.
Но opencode идентифицирует сессии по `project_id`, который не обновлялся.
В результате сессия не отображалась ни в старом, ни в новом проекте.

## Решение v2 (текущее)

### `core.py:354-398` — `update_session_directory`

1. **`_resolve_project_id(directory)`** — определяет `project_id` для новой директории:
   - Выполняет `git -C <dir> rev-parse --show-toplevel`
   - Ищет совпадение в таблице `project`
   - Если не найден — возвращает `global`

2. **UPDATE трёх полей:**
   ```sql
   UPDATE session SET directory = ?, path = ?, project_id = ? WHERE id = ?
   ```
   - `directory` — оригинальный путь с обратными слешами
   - `path` — нормализованный путь с прямыми слешами
   - `project_id` — git-хэш корня или `global`

3. **Каскад на subagent:**
   ```sql
   UPDATE session SET directory = ?, path = ?, project_id = ?
   WHERE parent_id = ?
   ```

### `app.py:1323-1364` — `_change_session_directory`

- Блокировка subagent: кнопка `disabled` при выборе дочерней сессии
- Диалог с подсказкой: «Переносите родительскую — дети переедут автоматически»
- Нормализация путей Windows: `tkinter` возвращает `/`, конвертируем в `\\`

## Сложности

1. **Не-git директории:** Если целевая папка не является git-репозиторием, `project_id` становится `global`. opencode может не показать такие сессии в боковой панели Desktop.
2. **Desktop-совместимость:** Desktop-приложение использует `.dat` файлы для UI-состояния (sessionTabs, воркспейсы). После переноса нужно вручную добавить сессию в эти файлы (см. `fix_desktop.py` в истории).
3. **Subagent orphan:** Если родительская сессия удалена, её дети становятся orphan'ами. Перенос такой сессии невозможен — нужно сначала восстановить родителя.
