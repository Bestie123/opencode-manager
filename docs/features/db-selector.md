# Выбор базы данных (DB Selector)

> Doc-ID: DB-SELECTOR-1 | Дата: 2026-06-02 | Связанные: [ARCH-v2], [S-MOVE-2]

## Проблема

У пользователя оказалось две SQLite-базы opencode:

| Файл | Размер | Сессий | Используется |
|------|--------|--------|-------------|
| `opencode.db` | 192 MB | 60 | Desktop-приложение (latest) |
| `opencode-dev.db` | 0.8 MB | 7 | CLI/TUI (dev-сборка) |

Менеджер был жёстко привязан к `opencode.db`. После автообновления opencode на dev-билд, CLI стал использовать `opencode-dev.db`, а менеджер по-прежнему показывал `opencode.db`.

## Решение

### Автоопределение (`core.py:90-116`)

`_detect_db_path()` теперь:
1. Сканирует все `opencode*.db` в `~/.local/share/opencode/`
2. Считает количество сессий в каждой
3. Возвращает БД с **максимальным числом сессий**

### Список БД (`core.py:118-142`)

`OpenCodeDB.list_databases()` — статический метод, возвращает `[(path, label, count), ...]` отсортированный по убыванию.

### Селектор в GUI (`app.py:125-131`)

- Combobox в тулбаре вкладки «Сессии»
- Отображает `имя (N сес.)`
- При переключении: создаёт `OpenCodeDB(path)` и перезагружает сессии
- Статус-бар показывает активную БД: `[opencode-dev] Загружено 7 сессий`

### Сложности

1. **PowerShell-обёртка:** `opencode` — это `.ps1` скрипт, не exe. `subprocess.run("opencode")` из Python падает с FileNotFoundError. Решение: вызывать через `powershell -NoProfile -Command "opencode db path"`.
2. **WAL-блокировка:** Если Desktop-приложение запущено, WAL-файлы могут блокировать чтение. Решение: таймаут `timeout=10` в `sqlite3.connect`.
3. **Бэкапы и копии:** В директории могут быть `opencode1.db`, `opencode - копия (2).db` — они тоже сканируются и показываются в списке.
