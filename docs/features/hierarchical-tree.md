# Иерархическое дерево сессий

> Doc-ID: H-TREE-1 | Дата: 2026-06-02 | Связанные: [DB-SELECTOR-1], [ARCH-v2]

## Проблема

В opencode-manager отображались ВСЕ сессии плоским списком, включая subagent-сессии.
opencode же показывает только **корневые** `(parent_id IS NULL)`, subagent-сессии скрыты.
Из-за этого пользователь видел 32 сессии в TestQA вместо 8.

## Решение

### 1. Treeview #0 column

`app.py:172-208` — переключен `show="headings"` → `show="tree headings"`.
Column #0 используется для названия сессии с Treeview-отступом.

### 2. Иерархическая вставка

`app.py:1033-1093` — `_refresh_tree()`:
1. Строит `children_of: dict[str, list[SessionInfo]]` — parent_id → [child, ...]
2. Вставляет корневые сессии на верхний уровень
3. Вставляет детей под родителя с `parent=iid`
4. Раскрывает родительские ноды (`self.tree.item(iid, open=True)`)

### 3. Колонка «Дочер.»

Для корневых — число детей. Для subagent — стрелка `→`.
Пусто, если детей нет.

### 4. Стилизация

```python
self.tree.tag_configure("subagent", foreground="#8b949e", font=("Consolas", 9, "italic"))
```

- Тёмная тема: `#8b949e` (серый)
- Светлая тема: `#6b7280`

### 5. Чекбокс «Sub»

`app.py:148-150` — `tk.BooleanVar(value=True)`. При выключении — показываются только корневые сессии (поведение как в opencode).

### 6. Поиск с родителем

`app.py:1027-1031` — если subagent совпал по поиску, его родитель тоже включается в результат.

### 7. Блокировка «Перенести в проект»

`app.py:222-229` — при выборе subagent кнопка `disabled`, при нажатии — диалог:
> «Это дочерняя сессия. Переносите родительскую — дети переедут автоматически.»

### Сложности

1. **Child-parent согласованность:** Если родитель отфильтрован поиском, дети всё равно показываются как корневые (чтобы не терять данные).
2. **Subagent без родителя:** 5 orphan-детей в `opencode.db` (родители удалены) — они отображаются как корневые.
